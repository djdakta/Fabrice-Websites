Page 'index.html'

La page 'index.html' est toujours la première page d'un site quand celui-ci est écrit en HTML! (dans 95% des cas!)
Quand on introduit l'URL (Unique Ressource Locator) d'un site dans la barre d'adresse du navigateur, par exemple: 'fabrice.the-abuzzers.com', celui-ci va automatiquement chercher/charger le fichier 'index.html' sans que le fichier ne soit renseigné dans cet URL! (ex.: 'fabrice.the-abuzzers.com/index.html')

Les espaces (retour à la ligne/lignes vides) en HTML ne font pas aller le texte à la ligne! Les espaces laissés entre les différentes lignes sont principalement pour 'nous' (les programmeurs) faciliter la tâche quand il sagit de devoir relire son code, ou quand il faut lire le code de quelqu'un d'autre!

Ici, il faut afficher le code source afin de pouvoir bien se rendre compte de ce qu'il se passe, et comprendre les informations qui sont dans les différents commentaires!

Mise en page:

La mise en page du fichier 'index.html' n'a ni queue ni tête mais ce n'est pas important étant donné que c'est notre tout premier site web!

Le principal ici, c'est que les balises principales soient utilisées, et, que notre premier site web statique fonctionne!

Oefening baart kunst!

A force d'écrire/coder des pages en 'html', on fini par connaître la majorité des balises par coeur, et écrire une page (balises comprises) ne prend plus plus de temps que si on tapait uniquement le texte sans les balises!

La numérotation des lignes lorsque le code source est affiché!? (clic droit n'importe où sur la page et choisir de voir le code source)

La numérotation des lignes s'avère très intéressante et nécessaire lorsque la page doit être 'débugged', ou que la page contient une/des erreurs!

Ici, on a un max de 35 lignes au total, lignes vide comprise, ce qui fait que si on devait retrouver une erreur ou peu importe, ça serait encore assez vite fait!
Par contre, quand une page a des milliers de lignes (ce qui n'est pas exceptionnel!), et qu'il faut retrouver 1 ligne bien précise, ben sans cette numérotation, c'est même pas la peine!

Le décalage entre les différentes lignes de code (tout n'est pas alligné à gauche) est principalement fait pour que le/les programmeurs voient clair dans le code grace aux différents "blocs".
Dans certains languages de programmation, les retraits sont nécessaire pour que le code fonctionne correctement mais c'est pas le cas dans l'HTML
